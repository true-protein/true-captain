# Reply Tone & Style Guide

## General Principles

- **Draft only, never send** — all replies are drafts for the user to review
- **No signature** — Outlook appends the user's signature automatically
- **Match the thread's tone** — if the sender is formal, reply formally; if casual, match that
- **Be concise** — executives value brevity; get to the point
- **Australian English** — use `organise`, `colour`, `recognise`, etc.

## External Communication (clients, partners, board, regulators)

- Professional and warm, not stiff
- Clear and direct — state the answer/action first, then context
- No jargon unless the sender used it first
- "Happy to" over "I would be delighted to"
- "Thanks" over "Thank you very much for your kind email"
- Avoid: excessive pleasantries, unnecessary apologies, filler phrases

### Example:
```
Hi Sarah,

Thanks for sending this through. I've reviewed the Q3 numbers — the margin
improvement looks solid.

Two things I'd flag:
- Freight costs are tracking 12% above forecast. Worth a closer look.
- The NetSuite reconciliation is off by $14k. I've asked the team to investigate.

Happy to jump on a call this week if you'd like to discuss.

[signature auto-appended by Outlook]
```

## Internal Communication (team, direct reports, peers)

- Direct and casual
- Skip pleasantries if the thread is already flowing
- Use dot points for multiple items
- OK to be brief — one-liner replies are fine internally

### Example:
```
Yep, approved. Go ahead.

Can you send me the updated forecast by Thursday? Want to review before the board pack goes out.
```

## Scheduling Replies

- Offer 3-5 time slots unless the user's config says otherwise
- Include day of week + date + time range
- State timezone explicitly if the recipient is external or in a different timezone
- If proposing on behalf of multiple people, note whose availability was checked

### Example:
```
Hi Sarah,

Here are some times that work on my end:

- Tuesday 11 Mar, 10:00–11:00 AM AEST
- Wednesday 12 Mar, 2:00–3:00 PM AEST
- Thursday 13 Mar, 11:00 AM–12:00 PM AEST

Let me know what suits and I'll send through a calendar invite.
```

## Things to Never Do

- Don't invent facts or figures — if unsure, say "I'll confirm and get back to you"
- Don't commit to deadlines on the user's behalf
- Don't accept or decline invitations without user approval
- Don't CC/BCC people the user hasn't specified
- Don't use emojis in external emails
