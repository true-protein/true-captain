# /setup — True Utilities Onboarding

## Overview

Set up your personal triage configuration interactively. This creates `~/.claude/triage-config.md` which all triage skills use.

If a config already exists, offer to update it rather than overwrite.

---

## Step 1: Check for Existing Config

Check if `~/.claude/triage-config.md` already exists.

- If it exists, read it and ask: "You already have a triage config. Want to update it or start fresh?"
- If it doesn't exist, proceed to Step 2.

---

## Step 2: Gather Information

Ask the user the following questions using AskUserQuestion. Group related questions together to keep it quick.

### Identity

Ask in one go:
- **What's your name?** (for draft replies)
- **What's your role?** (e.g. CFO, Head of Engineering, Marketing Manager — helps prioritise emails)
- **What's your email address?** (e.g. name@trueprotein.com.au)

### VIP Senders

Ask: **Whose emails should never be missed? List the email addresses of people whose messages are always high priority.**

Suggest common examples:
- CEO, board members, direct manager
- Key clients or external partners
- Regulators or auditors

The user can provide email addresses or just names/roles and you'll format them.

### Skip Patterns

Ask: **Are there any automated emails or notifications you want to always skip?**

Suggest common patterns:
- GitHub notifications (notifications@github.com)
- Jira/Confluence alerts
- Automated reports
- Marketing newsletters

### Scheduling Preferences

Ask using AskUserQuestion with options:
- **Work hours:** Default 9:00–17:00 AEST
- **Preferred meeting start:** Default 10:00 (avoid early mornings)
- **Time slots to offer in scheduling replies:** Default 3

### Communication Tone

Ask using AskUserQuestion with options:
- **External tone:** Professional and warm (default) / Formal / Casual
- **Internal tone:** Direct and casual (default) / Professional

### Channels

Ask using AskUserQuestion with multiSelect:
- **Which channels do you want included in /triage?**
  - Outlook email (default: yes)
  - Slack (default: yes)
  - Teams (default: yes)

---

## Step 3: Generate Config

Write the config file to `~/.claude/triage-config.md` using the gathered information.

Format it as:

```markdown
# Triage Configuration
#
# Generated by /setup on {date}
# Edit this file anytime, or run /setup again to reconfigure.

## Identity

- **Name:** {name}
- **Role:** {role}
- **Email:** {email}
- **Timezone:** Australia/Sydney

## VIP Senders (always action_required, always high priority)

{list of VIP email addresses, one per line with - prefix}

## Skip Patterns (auto-skip, don't even show in triage)

{list of skip patterns, one per line with - prefix}

## Scheduling Preferences

- **Work hours:** {start}–{end} AEST
- **Preferred meeting start:** {time}
- **Meeting buffer:** 15 min between meetings
- **Time slots to offer:** {count}
- **Weekdays only:** yes

## Communication Tone

- **External:** {external_tone}
- **Internal:** {internal_tone}

## Channels to Include in /triage

- Outlook email: {yes/no}
- Slack: {yes/no}
- Teams: {yes/no}
```

---

## Step 3.5: Inbox Hygiene Folders

After writing the config, check whether the user's email has the recommended folders for inbox sorting. The `/triage` skill moves skipped emails into categorised folders — but it can't create them (the MCP connectors don't support folder creation).

**For Outlook users:** Call `list-mail-folders` and `list-mail-child-folders` (for Inbox children) to check which folders exist.

**For Gmail users:** Call `gmail_list_labels` to check which labels exist.

Check for these recommended folders/labels. Also check Inbox subfolders (Outlook users often create folders inside Inbox). Accept common aliases as matches:

| Folder | Aliases also accepted | Purpose |
|--------|----------------------|---------|
| **Notifications** | Alerts | GitHub, Jira, Asana, Slack, automated alerts |
| **Marketing** | Newsletters, Promos | Newsletters, promos, vendor emails |
| **Calendar** | Meeting Responses | Calendar-related emails (responses, invites) — not the calendar app itself |
| **Reports** | Analytics, Dashboards | Automated reports, build alerts |
| **Payments** | Orders, Invoices, Billing | Invoices, receipts from billing systems |

Present the results:

- **If all folders exist:** "Your inbox folders are ready for triage sorting."
- **If some are missing:** List the missing ones and explain:

  > "The `/triage` skill sorts skipped emails into folders, but it can't create folders itself. These are missing from your mailbox:
  >
  > - **Notifications** — for GitHub/Asana/Slack alerts
  > - **Marketing** — for newsletters and promos
  >
  > Create them manually in Outlook/Gmail if you'd like finer sorting. Without them, `/triage` will move skipped emails to **Archive** instead (which still clears your inbox)."

- **If the email tool isn't available:** Skip this step silently.

Don't block setup on this — it's informational. The user can create folders later and `/triage` will pick them up on the next run.

## Step 4: Confirm

Show the user a summary of what was configured:

```
Setup complete! Here's your config:

- Name: {name} ({role})
- Email: {email}
- VIP senders: {count} configured
- Skip patterns: {count} configured
- Work hours: {hours}
- Channels: {list}

Config saved to ~/.claude/triage-config.md

You're ready to go! Try /triage to start.
```

---

## Notes

- Always use Australia/Sydney as the default timezone (True Protein is Australian)
- If the user gives partial info, use sensible defaults and tell them what was defaulted
- The config file includes comments explaining each section so users can edit it later
- If the user runs /setup again, preserve their existing VIP senders and skip patterns unless they explicitly want to change them
